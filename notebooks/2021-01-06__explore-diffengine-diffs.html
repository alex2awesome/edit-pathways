Because it seeks to maximize the overlap, it gets these garbled phrases. To fix this, we implemented a rolling-window merge operation. This gist of this algorithm is to:

  • While not done:
    • Iterate through each diff block, i in the text sequentially.
      • (where a diff block is a continuous block of characters sharing the same color: green, red or clear)
    • If diff block i+1 has length > window size w, merge

We develop a number of heuristics on how to merge with the contiguous blocks around depending on whether blocks i-1, i, i+1, i+2 are green, red or clear.

Sparing the gory details for the paper appendix, here is the output, with window w=20 characters. We use this merge algorithm going forward.
